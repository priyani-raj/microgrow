# MicroGrow

**Turn stolen lecture time into real CS skill — one 5-minute session at a time.**

MicroGrow is a phone-first, AI-powered learning companion built for CS students who spend hours in lectures they're not fully present for. Instead of scrolling, MicroGrow turns those scattered 5–20 minute windows into compounding progress — DSA practice, spaced-repetition flashcards, and quick concept explanations — all picked intelligently based on how much time you actually have.

---

## The Problem

Every CS student has sat through a lecture they weren't absorbing, phone in hand, doom-scrolling instead of studying. Existing tools (LeetCode, Anki, Notion) all assume you'll sit down with full focus and intent. None of them are built for **low-focus, interrupted, phone-only, 5-minute bursts** — which is where most students actually spend their "free" time.

MicroGrow is built specifically for that constraint.

## Features

### 🔐 Authentication
- Email/password registration and login
- JWT-based session management
- Protected routes on both frontend and backend

### 🧭 Onboarding
- New users select weak topics (Arrays, Trees, OS, DBMS, CN, System Design, etc.)
- Optional target companies for future personalization
- Drives which topics get prioritized in sessions

### ⏱️ Adaptive Session Mode (the core loop)
- User picks how much time they have: **5 / 10 / 15 / 20 minutes**
- The app automatically decides the right activity:
  - **≤ 6 min** → Flashcard review
  - **≤ 12 min** → Concept explanation
  - **> 12 min** → DSA problem
- Topic selection prioritizes the user's self-reported weak areas
- Live countdown timer during the session
- On completion: updates streak, total minutes, and session history

### 🤖 AI Concept Explainer
- Search any CS topic, get a ~60-second plain-language explanation with one analogy (powered by Groq)
- **"Still confused?" retry flow** — if the first explanation doesn't land, the AI generates a *completely different* analogy, simpler and jargon-free
- After a second unsuccessful attempt, the topic is flagged for deeper review later (rather than looping forever)
- Recent search history for quick re-access
- Quick-pick buttons for all core topics

### 🧩 AI DSA Problem Evaluator
- Pulls a random problem from the database (filtered by topic, with fallback to any topic if none match)
- User describes their **approach in plain English** — no compiler needed
- Groq evaluates the approach and returns:
  - A score out of 5
  - A verdict (Correct / Partially Correct / Incorrect / Off Track)
  - Estimated time complexity
  - 2–3 sentences of specific feedback
- Every attempt is saved for progress tracking

### 🃏 Flashcards with Spaced Repetition (SM-2)
- AI-generated flashcards (5 per topic) via Groq
- Classic **SM-2 algorithm** schedules reviews based on performance:
  - **Again** → resets interval, reviewed tomorrow
  - **Hard** → interval grows slowly
  - **Easy** → interval grows faster, ease factor increases
- Standalone Flashcards page to review due cards or generate new ones on demand
- Flip-to-reveal card UI

### 🧠 AI-Generated DSA Problem Bank
- Original problems generated by Groq across 9 core topics (Arrays, Linked Lists, Trees, Graphs, DP, OS, DBMS, CN, System Design) and all difficulty levels
- No copyright concerns — every problem is freshly AI-written, not scraped
- Bulk generator script (`generateProblems.js`) can be re-run anytime to expand the bank
- Each problem includes a title, prompt, expected approach (for AI evaluation), tags, and time estimate

### 🔥 Streaks
- Daily streak counter with grace logic (same-day sessions don't double-count, missed days reset the streak)
- Tracks current and longest streak

### 📊 Readiness Score & Progress Dashboard
- A 0–100 "Interview Readiness Score" computed from:
  - **40%** — topic coverage (how many core topics practiced)
  - **25%** — flashcard accuracy
  - **25%** — average DSA evaluation score
  - **10%** — streak consistency
- Dashboard shows: current streak, readiness score, total sessions/minutes, topics covered, weekly stolen-time stats, and a visual breakdown of the readiness score
- Progress page shows per-topic time breakdown and full session history log

### 🌙 Dark Mode
- System-aware by default, manually toggleable from the navbar
- Preference persists across sessions
- Applied across every page and component

## API Overview

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/auth/register` | Create account |
| POST | `/api/auth/login` | Log in, returns JWT |
| GET | `/api/auth/me` | Get current user |
| PUT | `/api/user/onboarding` | Save weak topics & target companies |
| PUT | `/api/user/settings` | Update profile |
| GET | `/api/session/start?minutes=` | Get the right activity for available time |
| POST | `/api/session/complete` | Log a completed session |
| GET | `/api/session/history` | Past sessions |
| POST | `/api/ai/explain` | 60-second concept explanation (with retry support) |
| POST | `/api/ai/evaluate` | Evaluate a DSA approach |
| POST | `/api/ai/generate-flashcards` | Generate flashcards for a topic |
| GET | `/api/flashcards/due` | Cards due for review |
| POST | `/api/flashcards/review` | Submit a review rating |
| GET | `/api/dsa/problems/random` | Get a random DSA problem |
| GET | `/api/progress` | Full dashboard stats & readiness score |

## Getting Started

### Backend
```bash
cd server
npm install
cp .env.example .env   # fill in MONGO_URI, JWT_SECRET, GROQ_API_KEY
node seed/seedProblems.js        # seed initial problems
node seed/generateProblems.js    # (optional) generate more with AI
npm run dev
```

### Frontend
```bash
cd client
npm install
npm run dev
```

The app runs on `http://localhost:5173` (frontend) and `http://localhost:5000` (backend).

## Why I Built This

I noticed that in every lecture, almost no one was actually paying attention — everyone, including me, was doom-scrolling. Rather than fight that, I built something that fits *into* that exact behavior: phone in hand, a few minutes at a time, zero pressure to focus deeply — but it compounds into real interview readiness over weeks.
